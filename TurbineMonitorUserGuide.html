  • Powerup system, connect wifi.

  • Open any web browser.
  • Enter the following IP address in the URL bar: 192.168.5.1

3. Status Indicator (Bottom Left of Figure 1)

  • Green Light: Communication normal. Receiving real-time data from the ECU.
  • Red Light: Communication lost. No valid data received for over 0.5 seconds. Check ECU power, wiring, or Wi-Fi connection.

Turbine Monitor technical reference manual

1. Serial Configuration

  • interface: RS232
  • Baud Rate: 19200 bps
  • Data Format: 8 Data bits, No Parity, 1 Stop bit

2. Packet Structure

All packets follow a fixed frame format:

  • Header: 0xFC 0xC5 (2 Bytes)
  • Packet Type: (1 Byte)
  • 0x52: Long Packet (Total 85 Bytes, Full Update)
  • 0x0C: Short Packet (Total 19 Bytes, RPM Update Only)
  • Payload: Variable
  • Footer: 0x5A (1 Byte)

2. Parsing Example (Long Packet - 0x52)

Taking the first line (85 Bytes) and applying the memory map:

  • [00 ~ 01] Header: FC C5
  • [02] Type: 52 (Long Packet)
  • [14 ~ 15] RPM: CA 00
  • Calculation: Hex 00 CA -> Decimal 202.
  • Actual RPM: 202 * 100 = 20,200 RPM
  • [22 ~ 24] Model: 4B 33 30 -> "K30"
  • [30 ~ 37] Status: 53 74 61 67 65 33 20 20 -> "Stage3 "
  • [46 ~ 52] Throttle: 54 68 3A 49 64 6C 65 -> "Th:Idle"
  • [54 ~ 59] VB: 20 39 2E 33 56 62 -> " 9.3Vb"
  • [62 ~ 66] VR: 20 36 2E 39 56 (+ 72 at offset 67) -> " 6.9Vr"
  • [70 ~ 74] Pump Power: 30 33 36 50 77 -> "036Pw"
  • [78 ~ 82] EGT: 33 34 37 B0 43 -> "347°C" * (Note: B0 is the ° symbol)
  • [84] Footer: 5A

3. Parsing Example (Short Packet - 0x0C)

Taking the second line (19 Bytes):

  • [00 ~ 01] Header: FC C5
  • [02] Type: 0C (Short Packet)
  • [14 ~ 15] RPM: CC 00
  • Calculation: Hex 00 CC -> Decimal 204.
  • Actual RPM: 204 * 100 = 20,400 RPM
  • [18] Footer: 5A
